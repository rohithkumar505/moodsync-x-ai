# MoodSync X AI — Deploy guide

## What goes where

| Part | Platform | Folder |
|------|----------|--------|
| **Frontend** (React) | **Vercel** | `frontend/` |
| **Backend** (Flask API) | **Render** | `backend/` |
| **Database** (Postgres) | **Render** (auto with blueprint) | — |
| **Mobile app** (Flutter) | App Store / Play Store (you build locally) | `mobile/` |

---

## 1. Backend on Render (do this first)

1. Push this repo to GitHub.
2. Open [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
3. Connect the repo — Render reads `render.yaml` at the repo root.
4. Wait for **moodsync-api** and **moodsync-db** to deploy.
5. Copy the backend URL, e.g. `https://moodsync-api.onrender.com`.
6. In Render → **moodsync-api** → **Environment**, set:
   - `CORS_ORIGINS` = your Vercel URL(s), e.g. `https://moodsync.vercel.app`
   - Optional: `OPENAI_API_KEY` for journal / assistant AI

Health check: `GET https://YOUR-API.onrender.com/api/health`

---

## 2. Frontend on Vercel

1. Open [Vercel](https://vercel.com) → **Add New Project** → import the same GitHub repo.
2. **Root Directory** → set to `frontend` (important).
3. Framework: **Vite** (auto-detected).
4. **Environment Variables**:
   - `VITE_API_URL` = `https://moodsync-api.onrender.com` (your Render URL, no trailing slash)
5. Deploy.

After deploy, add the live Vercel URL to Render `CORS_ORIGINS` if you did not set it yet.

---

## 3. Flutter mobile app

```bash
cd mobile
flutter pub get
# Edit lib/config/api_config.dart → set productionApiUrl to your Render URL
flutter run
```

Build release:

```bash
flutter build apk        # Android
flutter build ios        # iOS (Mac + Xcode)
```

Camera and music work on device when `productionApiUrl` points to your HTTPS Render API.

---

## Local development

```bash
# Terminal 1 — DB (optional Docker)
docker compose up -d db

# Terminal 2 — backend
cd backend && source venv/bin/activate && python app.py

# Terminal 3 — frontend
cd frontend && npm run dev
```

Frontend: `https://localhost:5173` · API proxied to `http://127.0.0.1:5001`

---

## Env reference

**Vercel (frontend)**  
- `VITE_API_URL` — Render backend URL

**Render (backend)**  
- `DATABASE_URL` — set automatically from Postgres  
- `SECRET_KEY`, `JWT_SECRET_KEY` — auto-generated by blueprint  
- `CORS_ORIGINS` — comma-separated Vercel URLs  
- `OPENAI_API_KEY` — optional
